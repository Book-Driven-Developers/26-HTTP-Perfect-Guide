HTTP가 인터넷 배달원이라면
HTTP 메시지는 무언가를 담아 보내는 소포와 같다.

어떻게 메시지를 만들고 이해하는지 알아본다.
## 3.1 메시지의 흐름
- HTTP 메시지: HTTP 애플리케이션 간에 주고받은 데이터의 블록들
- 클라이언트, 서버, 프록시 사이를 흐르며 방향이 있다. (인바운드, 아웃바운드, 업스트림, 다운스트림)

인바운드/아운바운드
- 트랜잭션의 방향을 표현
- 인바운드: 메시지가 서버로 향하는 것
- 아웃바운드: 메시지가 사용자 에이전트로 돌아오는것

업스트림/다운스트림
- 모든 메시지는 다운스트림으로 흐른다.

## 3.2 메시지의 각 부분
- 메시지는 단순한 데이터의 구조화된 블록이다.
- 구성: 시작줄, 헤더 블록, 본문
```
HTTP/1.0 200 OK

Content-Type: text/plain
Content-length: 19

Hi! I'm a message!
```
본문(Body)는 없을 수 있다.
lenght는 byte이다. 여기서는 19byte.

모든 메시지는 요청/응답 메시지로 구분된다.
**요청**
```
<메서드> <요청URL> <버전> # 시작줄
<헤더>                 # 헤더 블록

<엔터티 본문>            # 본문
```
**응답**
```
<버전> <상태코드> <사유 구절> # 시작줄
<헤더>                   # 헤더 블록

<엔터티 본문>              # 본문
```

**구성 요소**
- 메서드
- 요청 URL
- 버전
- 상태코드
- 사유 구절
- 헤더들
- 엔터티 본문

**시작줄**
- 요청줄
	- `<메서드> <요청URL> <버전>`
	- `GET /test/hi-ther.txt HTTP/1.1`
- 응답줄
	- `<버전> <상태코드> <사유 구절>`
	- `HTTP/1.0 200 OK`
- 메서드
	- GET, HEAD, POST, PUT, TRACE, OPTIONS, DELETE
- 상태코드
	- 1xx, 2xx, 3xx, 4xx, 5xx
	- 정보, 성공, 리다이렉션, 클라이언트 오류, 서버 오류
- 사유 구절
	- 상태 코드와 일대일 대응된다.
- 버전 번호
	- 1.1, 2.0, 3.0 등

**헤더**
- 일반 헤더
	- 요청/응답 공통 정보
- 요청 헤더
	- 요청에 대한 부가 정보
- 응답 헤더
	- 응답에 대한 부가 정보
- Entity 헤더
	- 본문 크기와 콘텐츠, 혹은 리소스 자체 서술
- 확장 헤더
	- 명세에 정의되지 않은 새로운 헤더

**엔터니 본문**
- 선택적이다. 없을 수 있음
- HTTP 메시지의 화물에 해당. 실제 내용물임.

## 3.3 메서드

안전한 메서드
- GET, HEAD 는 안전한 메서드.
- 실제 동작은 개발자에게 달렸다.

**GET**
- 조회 요청
- 본문이 담겨서 온다.

**HEAD**
- 응답으로 헤더만 돌려준다.
- 본문은 반환되지 않음.

**PUT**
- 서버에 수정을 요청한다.

**POST**
- 입력 데이터 전송을 위해 설계 되었다.

**TRACE**
- 트레이스 요청은 목적지 서버에서 '루프백' 진단을 시작한다.
	- 요청 전송의 마지막 단계에 있는 서버는 자신이 받은 요청 메시지를 본문에 넣어 TRACE 응답을 되돌려준다.
- 클라이언트는 자신과 목적지 서버 사이에 있는 모든 HTTP 애플리케이션의 요청/응답 연쇄를 따라가면서 자신이 보낸 메시지가 망가지거나/수정되었는지 어떻게 변경되었는지 확인한다.
- HTTP 애플리케이션은 메서드에 따라 다르게 동작한다.
	- 프록시는 GET이면 웹 캐시 같은 애플리케이션으로 POST 면 바로 서버로 전달 한다.
	- TRACE 요청을 어떻게 할지는 중간 애플리케이션이 결정한다.
- 요청 시 본문을 보낼 수 없다. 응답 본문에는 요청이 담긴다.

**OPTIONS**
- preflight 요청 시 사용됨.
- cors를 위해 서버가 지원하는 모든 리소스의 메서드를 확인.

**DELETE**
- 삭제한다.

## 3. 4 상태코드
1xx
- 정보성 상태 코드
2xx
- 성공 상태 코드
3xx
- 리다이렉션 상태 코드
	- 301: Moved Permanentyly
		- 영구적으로 이동되었을 때.
	- 307: Temporary Redirect
		- 임시로 이동된 경우
- Location 헤더를 담아서 응답한다.
4xx
- 클라이언트 에러 상태 코드
	- 401: 인증 실패
5xx
- 서버 에러 상태 코드
	- 505: HTTP Version Not Supported

## 3. 5 헤더
일반 헤더
- 둘다 사용하는 것.
	- Date: 날짜
	- Connection: 연결 옵션

요청 헤더
	- Accept
		- 클라이언트가 받고자 하는 데이터의 타입 정보
		- Chartset, Encoding, Language
	- Client iP, From, Host, Referer, User-Agent 등

응답 헤더
	- Server
		- 어떤 서버랑 대화중인지
	- Age: 요청이 얼마나 오래됐는지
	- Retry-After: 현재 리소스가 사용 불가능한 상태일 때.

엔터니 헤더
	- 엔터티 본문에 대한 헤더이다.
	- Content-Type
		- 본문 데이터 타입
		- 그 외에도 Encoding, Language, Lenght, Location, MD5, Range 있음
