# 5.2 간단한 펄 웹 서버

완전한 기능을 갖춘 HTTP 서버를 만들려면 해야 할 일이 많다.  
아파치 웹 서버의 코어는 **50,000줄이 넘는 코드**로 되어 있고, 부가 처리 모듈들을 더하면 훨씬 커진다.

HTTP/1.1의 기능들을 지원하려면 풍부한 리소스 지원, 가상 호스팅, 접근 제어, 로깅, 설정, 모니터링, 그 외 성능을 위한 각종 기능이 필요하다.  
그러나 **최소한으로 기능하는 HTTP 서버라면 30줄 이하의 펄(Perl) 코드로도 만들 수 있다.**

---

## type-o-serve: 진단용 최소 기능 웹 서버

예제로 제시된 **type-o-serve** 는 클라이언트와 프락시 간의 상호작용 테스트에 유용한 진단 도구이다.  
동작 방식은 다음과 같다.

1. 어느 웹 서버와 마찬가지로 HTTP 커넥션을 기다린다.
2. 요청 메시지를 받자마자 그 내용을 **화면에 출력**한다.
3. 그리고 관리자(사람)가 응답 메시지를 직접 **타이핑하기를 기다린다.**
4. 관리자가 마침표 하나뿐인 줄로 끝나는 응답을 입력하면, 그 응답을 클라이언트에게 보낸다.

이 방식 덕분에 HTTP 요청 메시지를 정확하게 기록하고, 어떤 HTTP 응답 메시지라도 손으로 만들어 돌려보낼 수 있다. (텔넷으로 요청을 직접 타이핑하던 것과 정반대로, 이번엔 응답을 직접 타이핑하는 셈이다.)

### type-o-serve 펄 코드

```perl
#!/usr/bin/perl
use Socket;
use Carp;
use FileHandle;

# (1) 명령줄에서 덮어쓰지 않는 이상 8080 포트를 기본으로 사용한다.
$port = (@ARGV ? $ARGV[0] : 8080);

# (2) 로컬 TCP 소켓을 생성하고 커넥션을 기다리도록(listen) 설정한다.
$proto = getprotobyname('tcp');
socket(S, PF_INET, SOCK_STREAM, $proto) || die;
setsockopt(S, SOL_SOCKET, SO_REUSEADDR, pack("l", 1)) || die;
bind(S, sockaddr_in($port, INADDR_ANY)) || die;
listen(S, SOMAXCONN) || die;

# (3) 시작 메시지를 출력한다.
printf("   <<<Type-O-Serve Accepting on Port %d>>>\n\n", $port);

while (1)
{
    # (4) 커넥션 C를 기다린다.
    $cport_caddr = accept(C, S);
    ($cport, $caddr) = sockaddr_in($cport_caddr);
    C->autoflush(1);

    # (5) 누구로부터의 커넥션인지 출력한다.
    $cname = gethostbyaddr($caddr, AF_INET);
    printf("   <<<Request From '%s'>>>\n", $cname);

    # (6) 빈 줄이 나올 때까지 요청 메시지를 읽어 화면에 출력한다.
    while ($line = <C>)
    {
        print $line;
        if ($line =~ /^\r/) { last; }
    }

    # (7) 응답 메시지를 위한 프롬프트를 만들고, 응답을 입력 받는다.
    #     "." 하나만으로 되어 있는 줄이 입력되기 전까지, 입력된 줄을 클라이언트에게 보낸다.
    printf("   <<<Type Response Followed by '.'>>>\n");
    while ($line = <STDIN>)
    {
        $line =~ s/\r//;
        $line =~ s/\n//;
        if ($line =~ /^\./) { last; }
        print C $line . "\r\n";
    }
    close(C);
}
```

### 사용 예시

80번 포트는 이미 다른 웹 서버가 쓰고 있으므로, 사용하지 않는 8080번 포트로 진단 서버를 띄운다.

```
% type-o-serve.pl 8080
```

브라우저로 `http://www.joes-hardware.com:8080/foo/bar/blah.txt` 에 접근하면, type-o-serve는 다음 요청 메시지를 화면에 그대로 출력한다.

```http
GET /foo/bar/blah.txt HTTP/1.1
Accept: */*
Accept-language: en-us
Accept-encoding: gzip, deflate
User-agent: Mozilla/4.0
Host: www.joes-hardware.com:8080
Connection: Keep-alive
```

관리자가 직접 다음 응답을 타이핑하면(마지막 `.` 으로 종료), 브라우저는 본문 `Hi there!` 를 출력한다.

```http
HTTP/1.0 200 OK
Connection: close
Content-type: text/plain

Hi there!
```

> **Java 개발자 관점**: 위 펄 코드의 `socket → bind → listen → accept` 흐름은 Java의 `ServerSocket` 과 완전히 동일한 추상화다.  
> `ServerSocket server = new ServerSocket(8080); Socket c = server.accept();` 한 줄 한 줄이 위 펄의 (2)~(4)에 대응한다. 차이는 "응답을 자동 생성하지 않고 사람이 손으로 타이핑한다"는 점뿐이며, 이 덕분에 어떤 비정상 응답도 재현해 클라이언트 동작을 테스트할 수 있다.
